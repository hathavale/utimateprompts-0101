# Learning & Education Prompts

A collection of prompts for accelerating learning, understanding complex topics, building new skills, and creating educational content.

---

## Explaining Concepts

### Explain a Complex Topic Simply

```
Explain [topic] to me as if I have no prior knowledge of [field].

Use simple, everyday language and avoid jargon. If you must use a technical term, define it immediately. Use an analogy or real-world example to make the core concept concrete.

Aim for clarity over completeness — give me just enough to understand the big picture.
```

### Use the Feynman Technique

```
Explain [concept] using the Feynman Technique:

1. Start with a plain-language explanation a 12-year-old could follow
2. Identify the parts most likely to be confusing or misunderstood
3. Re-explain those parts using a different analogy or example
4. Summarize the core idea in 2–3 sentences
```

### Explain the "Why"

```
I understand *what* [concept or process] is, but I don't understand *why* it works / *why* it matters.

Please explain:
1. The underlying reason or principle that makes it work
2. Why this approach was chosen over alternatives
3. What would happen if this concept or rule didn't exist
4. A concrete example that illustrates the "why"
```

### Explain Like I'm an Expert

```
Explain [topic] to me assuming I already have a strong background in [related field].

Skip the basics. Focus on:
- Nuances and edge cases
- Connections to advanced concepts in [related field]
- Common misconceptions among practitioners
- Current debates or open questions in this area
```

## Study & Skill Development

### Create a Learning Roadmap

```
Create a structured learning roadmap for mastering [skill or subject].

My current level: [beginner / intermediate / some specific background]
My goal: [what I want to be able to do when I'm done]
Time available: [hours per week]
Timeline goal: [e.g., "within 3 months"]

Provide:
1. A phased curriculum with milestones
2. Recommended resources for each phase (books, courses, projects)
3. Practice exercises to build skills progressively
4. How I'll know I've mastered each phase
```

### Generate Practice Problems

```
Generate [number] practice problems to help me master [skill or concept].

My current level: [beginner / intermediate]
Format: [multiple choice / short answer / coding exercise / case study]
Difficulty progression: [all the same level / gradually increasing]

After listing the problems, provide the answers or solutions separately so I can attempt them first.
```

### Create Flashcards

```
Create [number] flashcard pairs (question/answer) to help me memorize [topic].

Focus on:
- Key definitions and concepts
- Important distinctions between similar terms
- Common formulas, rules, or sequences

Format each card as:
Q: [question]
A: [answer]
```

### Identify Knowledge Gaps

```
I am studying [topic] and believe I understand the following:
[list what you think you know]

Please:
1. Identify likely gaps or misconceptions in my understanding
2. Ask me 3–5 diagnostic questions to test my knowledge
3. After I answer, tell me where my understanding is solid and where I need to go deeper
```

## Teaching & Instructional Design

### Create a Lesson Plan

```
Create a lesson plan for teaching [topic] to [audience, e.g., "high school students", "adult beginners", "professional developers"].

Duration: [lesson length, e.g., 60 minutes]
Prerequisites: [what learners should already know]
Learning objectives: by the end of this lesson, students will be able to [list 2–3 objectives]

Include:
1. An engaging opening / hook
2. Core content delivery with key teaching points
3. At least one interactive activity or exercise
4. A formative check for understanding
5. A summary and preview of what comes next
```

### Design a Quiz

```
Design a [number]-question quiz on [topic] for [audience].

Include a mix of:
- [number] multiple choice questions
- [number] true/false questions
- [number] short answer questions

For each question, indicate the correct answer and the concept being tested. Vary the difficulty from easy to challenging.
```

### Write an Explainer Article

```
Write an explainer article (approximately [word count] words) on [topic] for [target audience].

Structure:
1. An opening that explains why this topic matters
2. Core concepts explained clearly with examples
3. Common misconceptions to address
4. Practical takeaways or next steps
5. A brief conclusion

Use [formal / accessible / conversational] language appropriate for the audience.
```

## Mentorship & Feedback

### Give Constructive Feedback

```
I want to give constructive feedback to [person's role, e.g., "a junior developer", "a student", "a colleague"] on the following work:

[describe or paste the work]

Please help me structure feedback that:
1. Starts with genuine, specific praise for what is working well
2. Identifies 2–3 areas for improvement with concrete suggestions
3. Frames criticism as opportunities to grow rather than failures
4. Ends with encouragement and a clear next step
```

### Ask Socratic Questions

```
I want to help [person / myself] think through [topic or problem] more deeply using the Socratic method.

The current position or understanding is: [describe]

Generate a series of 5–8 probing questions that:
- Challenge assumptions without being dismissive
- Guide toward deeper understanding rather than giving answers
- Progress from surface-level to more fundamental questions
- Encourage the learner to articulate and examine their own reasoning
```

### Reflect on a Learning Experience

```
Help me reflect on a recent learning experience.

What I tried to learn / do: [describe]
What happened: [describe the outcome]
My initial reaction: [how I felt about it]

Please guide me through a structured reflection:
1. What did I actually learn (intended and unintended)?
2. What would I do differently next time?
3. What does this experience reveal about my current strengths and growth areas?
4. What is one concrete action I can take to build on this experience?
```
